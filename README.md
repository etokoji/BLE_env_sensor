# BLE環境センサー（最終版）

BLE（Bluetooth Low Energy）を使って環境センサー（温度・湿度・気圧）データを送信するESP32-C3用MicroPythonプロジェクト。

## プロジェクト概要

このプロジェクトは、ESP32-C3マイコンに接続された環境センサーからデータを取得し、BLEアドバタイジングを通じてデータを送信する低電力システムです。センサーデータはBLEアドバタイジングパケットの製造者固有データ（Manufacturer Specific Data）として埋め込まれ、受信側でデコードされます。

## 主な特徴

- **低電力設計**: ディープスリープとパワーオフ制御による長期間動作
- **コンパクトデータ**: 16バイトのバイナリ形式でセンサーデータを効率送信（照度含む）
- **マルチセンサー対応**: AHT10（温度・湿度）、BME280（気圧）、BH1750（照度）
- **エラーハンドリング**: センサー故障時のフォールバック機能
- **リーディングID管理**: RTCメモリを使った連続測定ID管理

## ファイル構成

### メインプログラム
- **`main.py`** - メインエントリーポイント、システム制御とスリープ管理
- **`read_sensors.py`** - センサーデータ取得とBLE送信処理

### BLE通信
- **`env_bluetooth_sender.py`** - BLEアドバタイジング送信クラス（現在使用中）
- **`bluetooth_sender.py`** - BLEサービス方式の送信クラス（旧版、現在未使用）

### センサードライバー
- **`AHT10.py`** - AHT10温度・湿度センサードライバー
- **`BME280.py`** - BME280温度・湿度・気圧センサードライバー
- **`bh1750.py`** - BH1750照度センサードライバー

### システム設定
- **`boot.py`** - MicroPython起動時設定（最小構成）

## ハードウェア構成

### ESP32-C3 ピン配置
- **GPIO 8**: SDA（I2Cデータ、AHT10 / BME280 / BH1750 共通バス）
- **GPIO 9**: SCL（I2Cクロック）
- **GPIO 0**: ADC入力（電源電圧監視）
- **GPIO 1**: ADC有効化（電圧分割抵抗制御）
- **GPIO 4**: 電源制御出力（DONE_PIN）

### 接続センサー
- **AHT10**: 温度・湿度センサー（I2Cアドレス: 0x38）
- **BME280**: 温度・湿度・気圧センサー（I2Cアドレス: 0x77）
- **BH1750**: 照度センサー（I2Cアドレス: 0x23、ADDRをVCCにすると0x5C）
- **電圧分割回路**: 電源電圧監視用

## データフォーマット

### BLEアドバタイジングデータ構造
```
Header: "ENV" (3 bytes)
Device ID: u8 (1 byte)
Reading ID: u16 (2 bytes, big-endian)
Temperature: i16 (2 bytes, 0.1°C単位)
Humidity: u16 (2 bytes, 0.1%単位) 
Pressure: u16 (2 bytes, 0.1hPa単位)
Voltage: u16 (2 bytes, 0.01V単位)
Illuminance: u16 (2 bytes, 1 lx 単位、符号なし。センサー未取得時は 0)

合計: 16 bytes（旧14バイトのみの実装とは受信側で互換分岐可能）
```

### データ精度
- 温度: ±0.1°C（符号付き）
- 湿度: ±0.1%
- 気圧: ±0.1hPa
- 電圧: ±0.01V
- 照度: ±1 lx（65535 lx まで）

## システム動作

1. **起動**: ハードリセット時は10秒待機、ディープスリープからの起動は即座に処理開始
2. **センサー読取り**: AHT10から温度・湿度、BME280から気圧、BH1750から照度、ADCから電源電圧を取得
3. **BLE送信**: 16バイトのバイナリデータをBLEアドバタイジングで110ms送信
4. **データ保存**: 送信成功時にReading IDをRTCメモリに保存
5. **パワーダウン**: 電源制御ピンをONにして外部電源カット、またはディープスリープ

## 設定パラメーター

### `main.py`
- `SleepTimeInMs`: スリープ時間（デフォルト: 19395ms ≈ 20秒）
- `DEV_ID`: デバイスID（デフォルト: 80）

### `env_bluetooth_sender.py`
- `BT_DEVICE_NAME`: BLEデバイス名（"EtoSense"）

### `read_sensors.py`
- `ADC_COF`: ADC変換係数（0.001090467）
- アドバタイジング時間: 110ms
- アドバタイジング間隔: 25ms

## エラーハンドリング

- **BME280センサー障害**: 気圧値を0.0として続行
- **BLE送信失敗**: Reading IDを保持して再送信を試行
- **RTCメモリ異常**: 初期値（Reading ID = 1）で復旧

## 電力管理

- **通常動作**: 約605ms（センサー読取り + BLE送信）
- **スリープ期間**: 19.4秒（設定可能）
- **外部電源制御**: DONE_PINによる電源カット制御

## セットアップ方法

1. ESP32-C3にMicroPythonファームウェアをインストール
2. 全てのPythonファイルをデバイスにアップロード
3. センサーとハードウェアを接続
4. `main.py`のパラメーターを必要に応じて調整
5. デバイスをリセットして動作開始

## 注意事項

- `bluetooth_sender.py`は旧バージョンのサービス方式実装のため現在未使用
- GPIO 2は電源投入時にHIGHになるためESP32-C3では使用不可
- I2C通信は10kHzの低速設定でセンサー安定性を確保
- RTCメモリは電源断で消去されるため外部電源制御時は要注意

## トラブルシューティング

- **センサー読取り失敗**: I2C配線とプルアップ抵抗を確認
- **BLE接続問題**: アドバタイジング間隔とデバイス名を確認
- **電源管理異常**: DONE_PINの配線と外部電源回路を確認
- **データ不整合**: Reading IDの範囲（0-65535）とオーバーフロー処理を確認

##  消費電力
- 約20秒周期で起動
- 消費電流: 4.2mAh
